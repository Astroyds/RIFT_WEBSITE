Governance and contribution

Contribution flow

  1. Discuss proposal in language RFC thread.
  2. Submit PR with syntax or docs evidence.
  3. Review by maintainers and governance council.
  4. Merge with release-note inclusion.

Governance model

RIFT maintains a steward + maintainer council model, with public change logs and versioned syntax decisions.

Roadmap timeline

  • Q1: parser stabilization
  • Q2: package tooling hardening
  • Q3: docs expansion + examples suite
  • Q4: LSP and editor integrations

Code of Conduct excerpt

Be respectful, collaborative, and evidence-driven. Technical disagreements should remain focused on implementation and measurable outcomes.
